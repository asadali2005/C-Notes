Section 4.1

Introduction to Fundamental Data Types

A variable is a named region of memory used to store information during a program's execution. To understand data types, you first need to understand how memory is physically organised.

Bits, Bytes & Memory Addresses

D
Bit
The smallest unit of memory. A bit holds exactly one of two values: 0 or 1. All data — numbers, text, images, programs — is ultimately a sequence of bits.
D
Byte
A group of bits operated on as a single unit. The modern standard defines 1 byte = 8 sequential bits. Memory is organised into sequential memory addresses, and each address holds exactly 1 byte.
F
Capacity Formula
An object using n bits can store 2n unique values.
Unique Values = 2n
Example: 1 byte = 8 bits → 28 = 256 unique values (0–255 unsigned, or −128 to +127 signed).
D
Data Type
A data type tells the compiler how to interpret the contents of memory in a meaningful way.
  • When you store a value, the compiler/CPU encodes it into the correct bit pattern.
  • When you read a value, the bit pattern is decoded back into the original value.
Without a data type the compiler cannot know whether a bit sequence represents an integer, a character, a boolean, etc.

Integer vs. Integral Types — A Critical Distinction

Standard Integer Types

Types specifically designed for whole numbers:

  • short
  • int
  • long
  • long long
  • Their signed/unsigned variants

Integral Types (Broader)

Types stored as integers internally, regardless of purpose:

  • bool
  • All char variants
  • All standard integer types

⚠️ enum is NOT considered integral even if stored as an integer.

ℹ
Nomenclature: The _t Suffix
Many modern types carry a _t suffix (e.g., std::nullptr_t, std::size_t). It simply stands for "type" — a naming convention signalling that the identifier is a type alias.

🗝 Section 4.1 Summary

  • Variables are named memory regions. Memory is organised in byte-sized addresses.
  • 1 byte = 8 bits. An n-bit object holds 2n unique values.
  • A data type tells the compiler how to interpret bit sequences.
  • C++ fundamental types: floating-point, integral (bool, char, integer), null pointer, void.
  • Integral is broader than integer — it includes bool and char types.
Section 4.2

Void

D
Void
void means "no type". It is an incomplete type — declared but not fully defined — so the compiler cannot determine how much memory to allocate. Incomplete types cannot be instantiated.

Uses of void

1. Functions that do not return a value

void_return.cpp 
#include <iostream>

// void return type means "return nothing"
void writeValue(int x) {
    std::cout << "The value of x is: " << x << '\n';
    // No return statement needed
}

int main() {
    writeValue(42);
    return 0;
}

2. Functions that take no parameters (Deprecated in C++)

❌ Old C-style (Avoid)

int getValue(void) { ... }

✅ Modern C++ (Preferred)

int getValue() { ... }
✓
Best Practice
Always use an empty parameter list () instead of (void) in C++.

🗝 Section 4.2 Summary

  • void = "no type" — an incomplete type that cannot be instantiated as a variable.
  • Used as a return type for functions that return nothing.
  • Do NOT write (void) for empty parameter lists in C++ — use ().
Section 4.3

Object Sizes and the sizeof Operator

Objects may use 1, 2, 4, 8, or more consecutive memory addresses. The number of bytes directly determines how many unique values the type can represent.

F
Capacity from Bytes
Unique Values = 2(bytes × 8)
1 byte → 256 values  |  2 bytes → 65,536 values  |  4 bytes → ~4.29 billion values
⚠
C++ Guarantees Minimums, NOT Exact Sizes
The C++ standard only specifies minimum sizes. Actual sizes depend on the compiler and target architecture.
  • A byte must be at least 8 bits.
  • char is exactly 1 byte (guaranteed).
  • short ≥ 16 bits  |  int ≥ 16 bits  |  long ≥ 32 bits  |  long long ≥ 64 bits

The sizeof Operator

D
sizeof
A unary operator that returns the size (in bytes) of a type or variable. Evaluated at compile time — no runtime cost. Return type is std::size_t.
sizeof_demo.cpp 
#include <iostream>

int main() {
    int x{};

    // sizeof with a type name — parentheses required
    std::cout << "bool:      " << sizeof(bool)      << " byte(s)\n";
    std::cout << "char:      " << sizeof(char)      << " byte(s)\n";
    std::cout << "short:     " << sizeof(short)     << " byte(s)\n";
    std::cout << "int:       " << sizeof(int)       << " byte(s)\n";
    std::cout << "long long: " << sizeof(long long) << " byte(s)\n";
    std::cout << "double:    " << sizeof(double)    << " byte(s)\n";

    // sizeof with a variable — parentheses optional but conventional
    std::cout << "x:         " << sizeof(x) << " byte(s)\n";

    return 0;
}

🗝 Section 4.3 Summary

  • Objects occupy 1 or more consecutive bytes. More bytes = more possible values.
  • C++ only guarantees minimum sizes — actual sizes are platform-dependent.
  • sizeof(type_or_var) returns the size in bytes at compile time, returning type std::size_t.
Section 4.4

Signed Integers

D
Integer
An integer is an integral type representing whole numbers (positive, negative, and zero). In C++, integers are signed by default — the sign is stored as part of the value.
Preferred FormVerbose Equivalents (Avoid)
shortshort int, signed short, signed short int

longlong int, signed long, signed long int

Signed Integer Ranges

F
Signed Integer Range Formula
−(2n−1)   to   2n−1 − 1
where n = number of bits. Half the patterns represent negatives; the other half represent zero and positives.
TypeTypical SizeMinimumMaximum
short16 bits (2 bytes)−32,76832,767

long32 or 64 bits≤ −2,147,483,648≥ 2,147,483,647

!
Signed Integer Overflow = Undefined Behavior
If an operation produces a value outside the range of a signed type, the result is undefined behaviour (UB). The program may crash, produce wrong results silently, or behave differently between compilers. Never allow it.

Integer Division — Truncation, Not Rounding

⚠
Integer Division Drops the Fraction
Dividing two integers always produces an integer result. The fractional part is truncated toward zero — it is NOT rounded.

8 / 5 → 1.6 → result: 1
-8 / 5 → −1.6 → result: −1 (truncated toward zero, not toward −2)
int_division.cpp 
#include <iostream>

int main() {
    std::cout <<  8 /  5 << '\n';   // 1  (1.6 truncated)
    std::cout << -8 /  5 << '\n';   // -1 (-1.6 truncated toward zero)
    std::cout <<  8 / -5 << '\n';   // -1 (-1.6 truncated toward zero)
    std::cout << -8 / -5 << '\n';   // 1  (1.6 truncated)
    return 0;
}

📝 Quiz — Signed Integers
Q1: What is the range of a 5-bit signed integer?
✅ Solution
Apply the formula with n = 5:

Min = −(24) = −16
Max = 24 − 1 = 15

Range: −16 to 15 (32 unique values = 25)
Q2a: What is the result of 13 / 5?
✅ Solution
Exact result: 2.6 → fraction dropped → 2
Q2b: What is the result of -13 / 5?
✅ Solution
Exact result: −2.6 → truncated toward zero → −2

🗝 Section 4.4 Summary

  • Signed integers: negative, zero, and positive whole numbers. Signed by default.
  • Range: −(2n−1) to 2n−1−1.
  • Signed integer overflow = undefined behavior.
  • Integer division truncates toward zero — it does NOT round.
Section 4.5

Unsigned Integers, and Why to Avoid Them

D
Unsigned Integer
An integer that holds only non-negative values (zero and positive). Declared with the unsigned keyword: unsigned int, unsigned short, etc.
F
Unsigned Range Formula
0   to   2n − 1
All bit patterns are used for non-negative values — the sign bit becomes an extra magnitude bit.
TypeBitsMinMax
unsigned short16065,535
unsigned int3204,294,967,295
unsigned long long64018,446,744,073,709,551,615

Unsigned Overflow — Modulo Wrapping

ℹ
Unsigned Overflow is Well-Defined (But Dangerous)
Unlike signed overflow (UB), unsigned overflow uses modulo wrapping (modulo 2n).

For an 8-bit unsigned (0–255):
  • 256 → wraps to 0
  • 257 → wraps to 1
  • −1 → wraps to 255
unsigned_wrap.cpp 
#include <iostream>

int main() {
    unsigned short x{ 0 };  // range: 0 to 65535
    x = x - 1;                 // 0 - 1 wraps to 65535!
    std::cout << x << '\n';   // prints: 65535
    return 0;
}

Why You Should Avoid Unsigned Integers for Quantities

!
Critical: Signed/Unsigned Mixing
When you mix a signed and an unsigned value in an expression, the signed value is implicitly converted to unsigned. If the signed value was negative, it wraps to a huge positive number — a silent, catastrophic bug.

unsigned int u { 2 }; signed int s { -1 };
std::cout << (s + u); → prints 4294967297 instead of 1!
✓
Best Practices
  • Favour signed numbers for general quantities, even non-negative ones.
  • Avoid mixing signed and unsigned in expressions.
  • Use unsigned only for bit manipulation, intentional wrap-around (cryptography), or specific low-level code.

🗝 Section 4.5 Summary

  • Unsigned integers: 0 to 2n−1. No negative values.
  • Overflow wraps via modulo 2n — well-defined but a frequent bug source.
  • Mixing signed + unsigned: signed is converted to unsigned → catastrophic if negative.
  • Default to signed integers for quantity values.

Fixed-Width Integers and std::size_t

Because int can be 16 or 32 bits depending on platform, C++11 introduced fixed-width integer types in <cstdint> that guarantee exact sizes everywhere.

Signed TypeUnsigned TypeExact SizeSigned RangeUnsigned Range
std::int8_tstd::uint8_t1 byte−128 to 1270 to 255
std::int16_tstd::uint16_t2 bytes−32,768 to 32,7670 to 65,535
std::int32_tstd::uint32_t4 bytes~−2.1B to ~2.1B0 to ~4.29B
std::int64_tstd::uint64_t8 bytes~−9.2×1018 to ~9.2×10180 to ~1.8×1019
!
Warning: 8-bit Fixed-Width Types Behave Like Chars!
Most compilers alias std::int8_t to signed char and std::uint8_t to unsigned char. They will print as characters, not numbers.

std::int8_t x { 65 }; std::cout << x; → prints A, not 65!

Avoid 8-bit fixed-width types for numeric values. Use static_cast<int>(x) if you must print them as numbers.

Fast and Least Types

std::int_fast#_t

The fastest type with at least # bits. The compiler chooses the most CPU-efficient native type.

std::int_least#_t

The smallest type with at least # bits. Prioritises minimal memory usage.

✓
Best Practice: Avoid Fast/Least Types
Their sizes vary across platforms, defeating the purpose of predictable sizing. Use explicit fixed-width types (std::int32_t, etc.) or plain int instead.

std::size_t

D
std::size_t
An implementation-defined unsigned integral type — the type returned by sizeof. Include <cstddef> to use it explicitly.
  • 32-bit systems: typically uint32_t
  • 64-bit systems: typically uint64_t
Its maximum value equals the maximum possible byte-size of a single object on that system. Widely used for array sizes, indices, and memory sizes.
sizet_demo.cpp 
#include <iostream>
#include <cstddef>   // for std::size_t

int main() {
    std::size_t s = sizeof(int);  // sizeof returns std::size_t
    std::cout << "Size of int: " << s << '\n';
    return 0;
}

🗝 Section 4.6 Summary

  • Fixed-width types (std::int32_t, etc.) from <cstdint> guarantee exact sizes on all platforms.
  • 8-bit fixed-width types behave like char — they print as characters. Avoid for numeric use.
  • Fast/least types have variable sizes — avoid them for portability.
  • std::size_t (from <cstddef>) is the unsigned type returned by sizeof.
Section 4.7

Introduction to Scientific Notation

D
Scientific Notation
A shorthand for very large or very small numbers: significand × 10exponent. In C++, use the letter e or E for the exponent.
1.2e4 = 1.2 × 104 = 12,000  |  5e-2 = 5 × 10−2 = 0.05

Significant Digits (Significant Figures)

ℹ
Rules for Counting Significant Digits
  • All non-zero digits are always significant.
  • Leading zeros are never significant (0.0042 → 2 sig digits).
  • Trailing zeros after a decimal point are significant (8.700 → 4 sig digits).
  • Trailing zeros with no decimal point are ambiguous / assumed insignificant (2100 → 2 sig digits assumed).

Floating Point Numbers

D
Floating Point
Data types that represent numbers with a fractional component. Called "floating point" because the decimal point can appear anywhere in the number. Floating-point types are always signed.
TypeSizePrecisionLiteral Suffix
float~4 bytes6–9 significant digitsf (e.g., 3.14f)
double~8 bytes15–18 significant digitsnone (default)
long double8, 12, or 16 bytes≥ 15 sig digitsL (e.g., 3.14L)
⚠
Floating-Point Literals Need a Decimal Point
5 → integer (int)  |  5.0 → double  |  5.0f → float  |  5.0L → long double

Precision, Output, and Rounding Errors

ℹ
Why Can't 0.1 Be Stored Exactly?
Computers store numbers in binary. Just as 1/3 cannot be exactly written in base 10, 0.1 (= 1/10) cannot be exactly expressed in binary. The closest approximation introduces tiny rounding errors that accumulate over calculations.
float_precision.cpp 
#include <iostream>
#include <iomanip>    // for std::setprecision()

int main() {
    float  f { 3.3333333333333333f };
    double d { 3.3333333333333333  };

    std::cout << std::setprecision(17);        // show 17 significant digits
    std::cout << "float:  " << f << '\n';   // 3.3333332538604736 (inaccurate after ~7 digits)
    std::cout << "double: " << d << '\n';   // 3.3333333333333335 (accurate to ~16 digits)
    return 0;
}
!
Never Compare Floating-Point Numbers with ==
Rounding errors mean two theoretically equal results may differ slightly.

if (0.1 + 0.2 == 0.3) may evaluate to false!

Use an epsilon (tolerance) comparison instead:
if (std::abs(result - expected) < 1e-9)

Also: never use floating-point for currency — use integer cents or a decimal library.
✓
Best Practices
  • Prefer double over float — better precision, small cost.
  • Avoid long double — platform-dependent, rarely worth it.
  • Never compare floats with ==.
  • Avoid division by 0.0 — produces Inf or NaN.

Special Values: NaN and Inf

ValueProduced byMeaning
Inf1.0 / 0.0Positive infinity
-Inf-1.0 / 0.0Negative infinity
NaN0.0 / 0.0, sqrt(-1.0)Not a Number — invalid arithmetic result
⚠
NaN Behaviour
Operations with NaN propagate NaN. Even NaN == NaN is false! Use std::isnan() and std::isinf() from <cmath> to check for them.

🗝 Section 4.8 Summary

  • Floating-point types represent fractional numbers. Always signed. Default literal type is double.
  • Use f suffix for float literals; no suffix for double.
  • Limited precision causes rounding errors — 0.1 is not exactly representable in binary.
  • Never use == to compare floats; never use for currency; avoid division by 0.0.
  • IEEE 754 special values: Inf, -Inf, NaN.
  • Prefer double over float.
Section 4.9

Boolean Values

D
bool
Holds exactly two logical values: true (stored as 1) or false (stored as 0). An integral type, typically 1 byte in size despite only needing 1 bit logically.
bool_demo.cpp 
#include <iostream>

int main() {
    bool b1 { true  };
    bool b2 { false };

    // Default output: 1 (true) or 0 (false)
    std::cout << b1 << '\n';  // prints: 1
    std::cout << b2 << '\n';  // prints: 0

    // Use std::boolalpha to print "true" / "false"
    std::cout << std::boolalpha;
    std::cout << b1 << '\n';  // prints: true
    std::cout << b2 << '\n';  // prints: false

    std::cout << std::noboolalpha;  // revert to 0/1 output
    return 0;
}

Boolean Input & Output

Output (std::cout)

  • Default: prints 1 or 0
  • std::boolalpha → prints "true" / "false"
  • std::noboolalpha → reverts to numeric

Input (std::cin)

  • Default: expects 0 or 1
  • std::cin >> std::boolalpha → accepts "true"/"false" (lowercase only)
  • Non-numeric input causes cin to silently fail
F
Integer → Boolean Conversion Rule
  • 0 → false
  • Any non-zero integer (including negatives) → true
So if (x) is equivalent to if (x != 0).

🗝 Section 4.9 Summary

  • bool holds true (= 1) or false (= 0).
  • cout prints 0/1 by default; std::boolalpha enables "true"/"false".
  • cin expects 0/1 by default; non-numeric input causes silent failure.
  • 0 → false; any non-zero integer → true.
Section 4.10

Introduction to if Statements

An if statement allows a program to execute code conditionally — only when a specified condition is true.

if_syntax.cpp 
// Basic if / else if / else structure
if (condition1) {
    // Executes only if condition1 is true
}
else if (condition2) {
    // Executes only if condition1 was false AND condition2 is true
}
else {
    // Executes if ALL prior conditions were false
}
ℹ
Non-Boolean Conditions
Any expression is implicitly converted to bool for evaluation. if (x) means "if x is non-zero". if (ptr) means "if the pointer is non-null".

Early Returns

D
Early Return
A return statement that appears before the last line of a function. When reached, it immediately stops the function and returns control (and optionally a value) to the caller.
early_return.cpp 
int absoluteValue(int x) {
    if (x < 0)
        return -x;   // Early return: exits immediately with -x

    return x;       // Only reached if x >= 0
}

Section 4.10 — Quiz Solutions

📝 Quiz — if Statements & Early Returns
Q1: What is an early return, and what is its behaviour?
✅ Solution
An early return is a return statement before the last line of a function. When executed, the function immediately stops and control returns to the caller. Any remaining code in the function is skipped. If the function has a non-void return type, the specified value is returned to the caller.
Q2: Write a program that asks the user for a digit 0–9, then prints whether it is prime (2, 3, 5, or 7).
✅ Solution
prime_digit.cpp 
#include <iostream>

bool isPrime(int x) {
    if (x == 2 || x == 3 || x == 5 || x == 7)
        return true;
    return false;
}

int main() {
    std::cout << "Enter a number 0 through 9: ";
    int x{};
    std::cin >> x;

    if (isPrime(x))
        std::cout << "The digit is prime\n";
    else
        std::cout << "The digit is not prime\n";

    return 0;
}
Q3: How can this code be shortened?

if (height >= 140.0) return true; else return false;
✅ Solution
The condition height >= 140.0 already produces a bool. Return it directly:
simplified_return.cpp 
return height >= 140.0;  // Directly returns the bool result of the comparison

🗝 Section 4.10 Summary

  • if / else if / else enables conditional execution. Conditions auto-convert to bool.
  • 0 = false; non-zero = true.
  • An early return immediately exits the function.
  • If a condition itself evaluates to bool, return it directly instead of wrapping in if/else.

Chars

D
char
A 1-byte integral type that holds a single character — a letter, digit, symbol, whitespace, or control character. Internally, char stores an integer that maps to a character via the ASCII encoding table.

ASCII Encoding

✅ Correct — Single Quotes for Chars

'a', '1', '!', '\n'

These are char literals.

❌ Wrong — Double Quotes for Single Chars

"a" is a string literal (const char*), not a char.
ℹ
std::cin and Whitespace
std::cin >> ch extracts one character but skips leading whitespace.
Use std::cin.get(ch) to capture the very next character, including spaces and newlines.
char_demo.cpp 
#include <iostream>

int main() {
    char c1 { 'a' };   // stores integer 97 internally
    char c2 { 97  };   // identical — also stores 97

    std::cout << c1 << '\n';                         // prints: a
    std::cout << c2 << '\n';                         // prints: a
    std::cout << static_cast<int>(c1) << '\n';       // prints: 97

    // Escape sequences in action
    std::cout << "Line 1\nLine 2\n";
    std::cout << "Col1\tCol2\n";
    std::cout << "She said \"hello\"\n";
    return 0;
}

Common Mistakes
  • Wrong slash direction: '/n' is a multicharacter literal (wrong!). '\n' is the newline escape (correct!).

  • Double quotes for chars: "a" is a string, not a char. Always use 'a'.

  • char: 1-byte integral type storing a character via its ASCII integer code.
  • Always use single quotes for char literals. Double quotes = string literal.
  • std::cin >> ch skips leading whitespace. std::cin.get(ch) captures any character.
  • Escape sequences begin with \. Key ones: \n, \t, \\, \', \".
  • Avoid multicharacter literals and wrong-direction slashes.
Section 4.12

Introduction to Type Conversion and static_cast

Implicit Type Conversion

D
Implicit Conversion
When the compiler automatically converts a value from one type to another without explicit instruction. Also called coercion. Occurs when passing mismatched types to functions, assigning between types, or mixing types in expressions.
⚠
Narrowing Conversions — Data Loss!
Converting from a more-precise type to a less-precise one (narrowing) can lose data.

double → int: fraction dropped. Compiler will warn.
Brace initialization blocks narrowing: int x { 5.5 }; is a compile error — use this as a safety feature.

Explicit Conversion: static_cast

D
static_cast
Tells the compiler to explicitly convert a value to a specified type. Checked at compile time. Silences warnings because you are explicitly acknowledging the conversion.
static_cast<new_type>(expression)
static_cast_demo.cpp 
#include <iostream>

int main() {
    // double -> int: intentionally drop fraction
    double d { 3.7 };
    int    i { static_cast<int>(d) };     // i = 3 (fraction dropped)
    std::cout << i << '\n';

    // char -> int: print ASCII code of a character
    char ch { 'a' };
    std::cout << static_cast<int>(ch) << '\n';   // prints: 97

    // int -> double: force floating-point division
    int x { 7 }, y { 2 };
    double result { static_cast<double>(x) / y };  // 7.0 / 2 = 3.5
    std::cout << result << '\n';               // prints: 3.5

    return 0;
}

Sign Conversions

ℹ
Signed ↔ Unsigned Conversions
Safe when the value is within range of the destination type. If out of range (C++20):
  • Signed → Unsigned out-of-range: modulo wrapping (defined in C++20).
  • Unsigned → Signed out-of-range: implementation-defined. Be careful.

📝 Quiz — Type Conversion & static_cast
Q1: Write a program where the user enters a character. Print it and its ASCII code using static_cast.
✅ Solution
char_ascii_explicit.cpp 
#include <iostream>

int main() {
    std::cout << "Enter a single character: ";
    char c{};
    std::cin >> c;

    std::cout << "You entered '"
              << c

🗝 Section 4.12 Summary

  • Implicit conversion: automatic. Widening is safe; narrowing may lose data and generates warnings.
  • Brace initialisation blocks narrowing conversions at compile time — use it for safety.
  • static_cast<T>(expr): explicit, compile-time-checked conversion. Preferred for intentional conversions.
  • Use static_cast<int> on a char to get its ASCII value.
  • Use static_cast<double> on an int to force floating-point division.

Q3: Simulate a ball dropped from a tower. Ask for height, then print ball position at 0–5 seconds. Formula: distance_fallen = 9.8 × t² / 2. Stop if height ≤ 0.
✅ Solution
falling_ball.cpp 
#include <iostream>

double getTowerHeight() {
    std::cout << "Enter the height of the tower in meters: ";
    double height{};
    std::cin >> height;
    return height;
}

// Kinematic formula: height = initial_height - (g * t^2 / 2)
double calculateHeight(double towerHeight, int seconds) {
    constexpr double gravity { 9.8 };
    double fallen { gravity * (seconds * seconds) / 2.0 };
    return towerHeight - fallen;
}

void printHeight(double height, int seconds) {
    if (height > 0.0)
        std::cout << "At " << seconds
                  << " seconds, the ball is at height: "
                  << height << " meters\n";
    else
        std::cout << "At " << seconds
                  << " seconds, the ball is on the ground.\n";
}

int main() {
    double towerHeight { getTowerHeight() };

    printHeight(calculateHeight(towerHeight, 0), 0);
    printHeight(calculateHeight(towerHeight, 1), 1);
    printHeight(calculateHeight(towerHeight, 2), 2);
    printHeight(calculateHeight(towerHeight, 3), 3);
    printHeight(calculateHeight(towerHeight, 4), 4);
    printHeight(calculateHeight(towerHeight, 5), 5);

    return 0;
}

/*
 * Physics recap:
 * distance_fallen = ½ × g × t²
 * height_of_ball  = initial_height - distance_fallen
 *
 * t=0: fallen=0 m       → height = towerHeight
 * t=1: fallen=4.9 m     → height = towerHeight - 4.9
 * t=2: fallen=19.6 m    → height = towerHeight - 19.6
 * t=3: fallen=44.1 m    → etc.
 */

★ Master Summary — Chapter 4 Revision Sheet

Everything you need to remember from Chapter 4 at a glance. Use this as your final review before any test.

Memory Basics

  • 1 byte = 8 bits (sequential)
  • n bits → 2n unique values
  • Data type = how bits are interpreted
  • sizeof(T) → size in bytes, returns std::size_t

void

  • Means "no type" — incomplete, non-instantiable
  • Return type for value-less functions
  • Use () not (void) for empty param lists

Signed Integers

  • Range: −(2n−1) to 2n−1−1
  • Overflow = undefined behavior
  • Division truncates toward zero (no rounding)
  • Prefer: short, int, long, long long

Unsigned Integers

  • Range: 0 to 2n−1
  • Overflow wraps (modulo 2n)
  • Mixing with signed → signed becomes unsigned (dangerous if negative)
  • Prefer signed for general quantities

Fixed-Width Integers

  • #include <cstdint>
  • std::int8_t through std::int64_t
  • 8-bit types act like chars — avoid for numbers
  • std::size_t from <cstddef>, unsigned, for sizes

Scientific Notation

  • Format: significand × 10exp → C++: 1.5e4
  • Leading zeros: never significant
  • Trailing zeros after decimal: significant
  • Trailing zeros, no decimal: ambiguous (assumed insignificant)

Floating Point

  • float: 4 bytes, ~6–9 sig digits
  • double: 8 bytes, ~15–18 sig digits (prefer)
  • Default literal type: double. Suffix f for float
  • Rounding errors exist; 0.1 ≠ exact in binary
  • Never use ==; never for currency; avoid ÷ 0.0
  • Special values: Inf, -Inf, NaN

Boolean

  • true = 1, false = 0
  • cout: 0/1 by default; std::boolalpha for words
  • cin: expects 0/1 by default
  • 0 → false; any non-zero → true

if Statements & Early Return

  • Condition auto-converts to bool
  • 0 = false, non-zero = true
  • Early return: immediately exits function
  • Return boolean expression directly: return x >= 0;

Chars

  • 1 byte, stores integer (ASCII code)
  • Single quotes for literals: 'a' not "a"
  • cin >> ch: skips whitespace; cin.get(ch): doesn't
  • Escape sequences: \n \t \\ \' \"
  • Avoid multichar literals: 'ab' or '/n'

Type Conversion

  • Implicit: automatic by compiler
  • Narrowing loses data → compiler warns
  • Brace-init blocks narrowing at compile time
  • static_cast<T>(expr): explicit, safe, preferred
  • char→int: get ASCII value; int→double: force fp division

Critical Best Practices

  • Prefer double over float
  • Prefer signed over unsigned for quantities
  • Never compare floats with ==
  • Use static_cast for intentional conversions
  • Avoid long double and 8-bit fixed-width types
  • Use () not (void) for empty params

Key Formulas at a Glance

  • Capacity: 2n unique values from n bits
  • Signed range: −(2n−1) to 2n−1−1
  • Unsigned range: 0 to 2n−1
  • 5-bit signed: −16 to 15  |  5-bit unsigned: 0 to 31
  • Falling ball: height = towerH − (9.8 × t² / 2)
  • static_cast: static_cast<new_type>(expression)
